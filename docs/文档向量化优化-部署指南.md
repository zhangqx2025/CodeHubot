# 文档向量化优化 - 部署指南

本文档说明了新增的两个核心功能：
1. **预览文档切分结果** - 向量化前查看和确认
2. **消息队列处理** - 使用 Celery + Redis 确保稳定性

## 📋 功能概述

### 1. 预览文档切分

- 上传前可以预览文档如何被切分成文本块
- 查看切分摘要（文本块数量、大小、切分方式等）
- 浏览每个文本块的内容预览
- 对切分结果不满意可以返回调整参数
- 确认无误后再进行向量化

**使用流程：**
```
上传文档 → 选择切分方式 → 预览切分 → 确认 → 向量化
```

### 2. 消息队列处理

- 使用 Celery + Redis 替代简单的后台任务
- 任务持久化，服务重启不丢失
- 自动重试机制（失败最多重试3次）
- 更好的并发控制和资源管理
- 任务状态追踪

## 🚀 快速开始

### 1. 安装 Redis

**macOS:**
```bash
brew install redis
brew services start redis
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt update
sudo apt install redis-server
sudo systemctl start redis
sudo systemctl enable redis
```

**验证 Redis 是否运行：**
```bash
redis-cli ping
# 应返回: PONG
```

### 2. 配置环境变量

编辑 `backend/.env` 文件，确保包含：
```env
# Redis 配置
REDIS_URL=redis://localhost:6379

# 阿里云 DashScope API 密钥（向量化必需）
DASHSCOPE_API_KEY=sk-your-actual-api-key-here
```

### 3. 启动 Celery Worker

**开发环境（前台运行）：**
```bash
cd backend

# 方式1: 使用 celery 命令
celery -A app.core.celery_app worker --loglevel=info --concurrency=4

# 方式2: 使用 Python 脚本
python celery_worker.py worker --loglevel=info
```

**生产环境（后台运行）：**
```bash
cd backend

# 启动 Worker
bash scripts/start_celery_worker.sh

# 查看日志
tail -f logs/celery_worker.log

# 停止 Worker
bash scripts/stop_celery_worker.sh
```

### 4. 启动后端服务

```bash
cd backend
python main.py
```

### 5. 启动前端服务

```bash
cd frontend
npm run dev
```

## 📖 使用指南

### 预览文档切分

1. **进入知识库详情页**
   - 点击"上传文档"按钮

2. **配置切分参数**
   - 选择文件
   - 设置文档标题（可选）
   - 选择切分方式：
     - 固定大小（默认500字符）
     - 按段落（双换行）
     - 按句子
     - 自定义大小

3. **预览切分结果**
   - 点击"预览切分"按钮
   - 查看切分摘要和参数
   - 浏览文本块列表
   - 点击"查看全文"查看完整内容

4. **调整或确认**
   - 如果不满意：点击"返回修改"，调整参数后重新预览
   - 如果满意：点击"确认上传并向量化"

5. **等待处理**
   - 文档上传成功后，向量化任务自动提交到队列
   - 可在文档列表查看状态（待处理 → 处理中 → 已完成）

### 切分方式选择建议

| 切分方式 | 适用场景 | 优点 | 缺点 |
|---------|---------|------|------|
| **固定大小** | 通用文档、代码文件 | 大小均匀、Token可控 | 可能截断句子或段落 |
| **按段落** | 结构化文档、博客文章 | 语义完整 | 大小不均匀 |
| **按句子** | 短文本、对话记录 | 句子完整 | 文本块较小 |
| **自定义** | 大型文档、特殊需求 | 灵活控制 | 需要手动设置 |

### 文本块数量参考

- **< 50 块**: 小型文档，快速处理
- **50-100 块**: 中型文档，正常处理
- **100-200 块**: 大型文档，需要较长时间
- **> 200 块**: 超大文档，建议拆分或增大块大小

## 🔧 Celery Worker 管理

### 查看 Worker 状态

```bash
# 查看活跃的 Worker
celery -A app.core.celery_app inspect active

# 查看已注册的任务
celery -A app.core.celery_app inspect registered

# 查看 Worker 统计信息
celery -A app.core.celery_app inspect stats
```

### 查看任务队列

```bash
# 查看等待中的任务
celery -A app.core.celery_app inspect reserved

# 查看活跃任务
celery -A app.core.celery_app inspect active

# 查看任务统计
celery -A app.core.celery_app inspect stats
```

### 控制 Worker

```bash
# 停止所有 Worker
celery -A app.core.celery_app control shutdown

# 重启 Worker（需要进程管理器，如 systemd）
systemctl restart celery-worker

# 清空队列（慎用）
celery -A app.core.celery_app purge
```

## 🐳 Docker 部署

### docker-compose.yml 配置

```yaml
version: '3.8'

services:
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    restart: always

  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - REDIS_URL=redis://redis:6379
      - DASHSCOPE_API_KEY=${DASHSCOPE_API_KEY}
    depends_on:
      - redis
      - mysql
    restart: always

  celery_worker:
    build: ./backend
    command: celery -A app.core.celery_app worker --loglevel=info --concurrency=4
    environment:
      - REDIS_URL=redis://redis:6379
      - DASHSCOPE_API_KEY=${DASHSCOPE_API_KEY}
    depends_on:
      - redis
      - mysql
    restart: always

  flower:
    build: ./backend
    command: celery -A app.core.celery_app flower --port=5555
    ports:
      - "5555:5555"
    environment:
      - REDIS_URL=redis://redis:6379
    depends_on:
      - redis
      - celery_worker
    restart: always

volumes:
  redis_data:
```

### 启动服务

```bash
docker-compose up -d
```

## 🔍 故障排查

### 1. Celery Worker 无法连接 Redis

**症状：**
```
ERROR: Cannot connect to redis://localhost:6379
```

**解决：**
```bash
# 检查 Redis 是否运行
redis-cli ping

# 检查环境变量
echo $REDIS_URL

# 如果 Redis 在其他主机，更新 .env
REDIS_URL=redis://your-redis-host:6379
```

### 2. 任务一直处于待处理状态

**症状：**
- 文档状态一直是"待处理"
- 任务没有被 Worker 消费

**原因：**
- Worker 未启动
- Worker 崩溃
- 队列阻塞

**解决：**
```bash
# 检查 Worker 是否运行
ps aux | grep celery

# 查看 Worker 日志
tail -f logs/celery_worker.log

# 重启 Worker
bash scripts/stop_celery_worker.sh
bash scripts/start_celery_worker.sh
```

### 3. 向量化失败

**症状：**
```
ERROR: 需要提供通义千问API密钥
```

**解决：**
```bash
# 检查环境变量
echo $DASHSCOPE_API_KEY

# 确保 .env 文件包含
DASHSCOPE_API_KEY=sk-your-actual-key

# 重启 Worker（让新环境变量生效）
bash scripts/stop_celery_worker.sh
bash scripts/start_celery_worker.sh
```

### 4. 预览切分失败

**症状：**
- 点击"预览切分"后报错
- "文件解析失败"

**常见原因：**
- 文件格式不支持（只支持 TXT 和 MD）
- 文件编码问题
- 文件过大（>5MB）
- 文件为空

**解决：**
- 确保文件是 `.txt` 或 `.md` 格式
- 确保文件使用 UTF-8 编码
- 大文件可以先拆分
- 检查文件内容不为空

### 5. Worker 资源占用过高

**症状：**
- CPU 或内存占用过高
- 处理速度变慢

**解决：**
```bash
# 降低并发数（修改启动脚本）
celery -A app.core.celery_app worker --concurrency=2

# 限制每个 Worker 处理的任务数
celery -A app.core.celery_app worker --max-tasks-per-child=20
```

## ⚙️ 性能优化

### 1. 调整并发数

根据服务器 CPU 核心数调整：
```bash
# 推荐: CPU 核心数 + 1
celery -A app.core.celery_app worker --concurrency=5
```

### 2. 限制任务时间

防止任务卡死：
```python
# backend/app/core/celery_app.py
celery_app.conf.update(
    task_time_limit=3600,      # 硬限制1小时
    task_soft_time_limit=3000  # 软限制50分钟
)
```

### 3. 批量处理

如果需要批量上传文档，可以：
- 使用脚本批量提交任务
- Worker 会自动排队处理
- 通过 Flower 监控进度

## 📝 开发者指南

### 添加新的 Celery 任务

1. 在 `backend/app/tasks/` 创建新的任务文件
2. 定义任务函数：

```python
from app.core.celery_app import celery_app

@celery_app.task(name='my_task', bind=True, max_retries=3)
def my_task(self, param1, param2):
    try:
        # 任务逻辑
        result = do_something(param1, param2)
        return {'status': 'success', 'result': result}
    except Exception as e:
        # 重试机制
        if self.request.retries < self.max_retries:
            raise self.retry(exc=e, countdown=60)
        return {'status': 'error', 'message': str(e)}
```

3. 在 API 中调用：

```python
from app.tasks.my_tasks import my_task

@router.post("/my-endpoint")
async def my_endpoint():
    task = my_task.delay(param1, param2)
    return {'task_id': task.id}
```

## 🚦 生产环境建议

### 1. 使用进程管理器

**systemd 配置 (celery-worker.service):**
```ini
[Unit]
Description=Celery Worker for CodeHubot
After=network.target redis.service

[Service]
Type=forking
User=www-data
Group=www-data
WorkingDirectory=/path/to/CodeHubot/backend
Environment="PATH=/path/to/venv/bin"
ExecStart=/path/to/venv/bin/celery -A app.core.celery_app worker \
    --loglevel=info \
    --concurrency=4 \
    --pidfile=/var/run/celery/worker.pid \
    --logfile=/var/log/celery/worker.log \
    --detach
ExecStop=/path/to/venv/bin/celery -A app.core.celery_app control shutdown
Restart=always

[Install]
WantedBy=multi-user.target
```

启用服务：
```bash
sudo systemctl enable celery-worker
sudo systemctl start celery-worker
```

### 2. 日志轮转

创建 `/etc/logrotate.d/celery`：
```
/var/log/celery/*.log {
    daily
    rotate 14
    compress
    delaycompress
    notifempty
    create 0644 www-data www-data
    sharedscripts
    postrotate
        systemctl reload celery-worker > /dev/null
    endscript
}
```

### 3. Redis 持久化

编辑 `/etc/redis/redis.conf`：
```conf
# 启用 AOF
appendonly yes
appendfilename "appendonly.aof"
appendfsync everysec

# RDB 快照
save 900 1
save 300 10
save 60 10000
```

- 监控 Redis 内存使用
