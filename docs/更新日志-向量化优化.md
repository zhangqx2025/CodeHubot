## ✨ 新增功能

### 1. 预览文档切分

#### 功能描述

上传文档时，可以先预览文档将如何被切分成文本块，确认无误后再进行向量化处理。

#### 主要特性

- **切分预览**：查看文档切分后的所有文本块
- **摘要信息**：显示文本块数量、大小、切分方式等
- **内容预览**：浏览每个文本块的内容（支持查看全文）
- **参数调整**：对切分结果不满意可返回修改参数
- **编码显示**：显示检测到的文件编码和置信度
- **智能警告**：文本块过多时提供优化建议

#### 使用流程

```
选择文件 → 配置切分参数 → 点击"预览切分" 
  ↓
查看切分结果（编码、文本块数、内容预览）
  ↓
满意？
├─ 是 → 点击"确认上传并向量化"
└─ 否 → 点击"返回修改" → 调整参数 → 重新预览
```

#### API 端点

```
POST /api/kb-documents/{kb_uuid}/preview
```

**参数：**
- `file`: 文件（必需）
- `split_mode`: 切分方式（fixed/paragraph/sentence/custom）
- `chunk_size`: 块大小（仅 custom 模式）
- `chunk_overlap`: 重叠大小（仅 custom 模式）

**返回：**
```json
{
  "code": 200,
  "message": "文档切分预览成功（编码: utf-8）",
  "data": {
    "file_name": "example.txt",
    "file_size": 12345,
    "file_type": "txt",
    "file_encoding": "utf-8",
    "encoding_confidence": "99.0%",
    "content_stats": {
      "total_chars": 5000,
      "chinese_chars": 3500,
      "english_chars": 1000
    },
    "total_chunks": 12,
    "preview_chunks": [...],
    "split_config": {
      "split_mode": "fixed",
      "chunk_size": 500,
      "chunk_overlap": 50
    }
  }
}
```

### 2. Celery 消息队列处理

#### 功能描述

使用 Celery + Redis 替代简单的后台任务，提供更稳定、更可靠的异步处理能力。

#### 主要特性

- **任务持久化**：任务存储在 Redis 中，服务重启不丢失
- **自动重试**：失败任务自动重试（最多 3 次，间隔 60 秒）
- **并发控制**：可配置 Worker 并发数，优化资源利用
- **任务追踪**：每个任务有唯一 ID，可追踪执行状态
- **优雅降级**：Celery 不可用时自动回退到后台任务
- **进程管理**：支持 systemd、supervisor 等进程管理
- **监控支持**：可集成 Flower 进行 Web 监控

#### 架构变化

**之前（FastAPI BackgroundTasks）：**
```
用户上传 → 后台任务 → 向量化
```
- ❌ 服务重启任务丢失
- ❌ 无法重试失败任务
- ❌ 并发控制有限

**现在（Celery + Redis）：**
```
用户上传 → Redis 队列 → Celery Worker → 向量化
```
- ✅ 任务持久化
- ✅ 自动重试机制
- ✅ 灵活的并发控制
- ✅ 任务状态追踪
- ✅ 分布式支持

**后端：**

### 3. 智能文件编码处理

#### 功能描述

自动检测和处理各种文件编码，避免乱码问题，支持 UTF-8、GBK、GB2312、GB18030、BIG5 等常见编码。

#### 主要特性

- **自动编码检测**：使用 chardet 库智能检测文件编码
- **多编码尝试**：置信度低时依次尝试常见编码
- **智能回退**：所有编码失败时使用 UTF-8 + replace
- **乱码检测**：自动检测乱码比例，超过 10% 时报错
- **编码显示**：预览界面显示检测到的编码和置信度
- **编码警告**：非 UTF-8 编码时显示警告和转换建议
- **BOM 处理**：自动去除 UTF-8 BOM 标记

#### 编码检测流程

```
读取文件
  ↓
chardet 检测编码
  ↓
置信度 ≥ 70%？
├─ 是 → 使用检测编码
└─ 否 → 尝试常见编码（UTF-8 → GBK → GB2312 → GB18030 → BIG5）
    ↓
解码成功？
├─ 是 → 继续处理
└─ 否 → UTF-8 + replace
    ↓
检查乱码比例（�）
    ↓
> 10%？
├─ 是 → 抛出错误（要求用户转换编码）
└─ 否 → 继续处理
```

#### 支持的编码

| 编码 | 支持程度 | 说明 |
|------|---------|------|
| UTF-8 | ✅ 完全支持 | 推荐使用 |
| GBK | ✅ 完全支持 | Windows 中文默认 |
| GB2312 | ✅ 完全支持 | 简体中文 |
| GB18030 | ✅ 完全支持 | 国标编码 |
| BIG5 | ✅ 完全支持 | 繁体中文 |
| Latin-1 | ⚠️ 基本支持 | 西欧语言 |
| CP1252 | ⚠️ 基本支持 | Windows 西欧 |

## 🔄 功能改进

### 文档上传流程优化

**之前：**
```
选择文件 → 配置参数 → 上传 → 自动向量化
```
- ❌ 无法预览切分结果

**现在：**

- ✅ 提前查看切分效果
- ✅ 编码问题立即发现
- ✅ 不满意可返回修改
- ✅ 确认无误再提交

### 向量化任务处理优化

**之前：**
```python
# 简单的后台任务
background_tasks.add_task(run_document_embedding, doc_id)
```

**现在：**
```python
# Celery 任务（带重试）
task = embed_document_task.delay(doc_id)
logger.info(f"任务ID: {task.id}")
```

优势：
- 任务持久化存储
- 自动重试机制（3 次，间隔 60 秒）
- 可追踪任务状态
- 服务重启任务不丢失

### 错误处理优化

**编码错误提示：**

之前：
```

```

现在：
```

请确保文件使用 UTF-8、GBK 或 GB2312 编码。

建议：请使用 UTF-8 编码保存文件，或使用"记事本"等工具转换编码后重试。
```

## ⚠️ 注意事项

### Redis 依赖

- **必需**：Celery 需要 Redis 作为 broker 和 backend
- **可选**：可以使用其他 broker（如 RabbitMQ），但推荐 Redis
- **配置**：确保 `REDIS_URL` 正确

### Worker 管理

- **开发环境**：前台运行，Ctrl+C 停止
- **生产环境**：使用 systemd 或 supervisor 管理
- **监控**：建议使用 Flower 进行 Web 监控

### 编码建议

- **强烈推荐**：统一使用 UTF-8 编码
- **兼容性**：GBK 等编码可以处理，但建议转换
- **最佳实践**：上传前使用预览功能检查编码

### 性能调优

- **并发数**：根据 CPU 核心数调整（建议 核心数 + 1）
- **内存**：每个 Worker 约占用 200-500MB
- **Redis**：建议配置持久化（AOF + RDB）

## 🆘 故障排查

### 预览失败

**症状：** 点击"预览切分"后报错

**检查：**
1. 文件格式是否支持（仅 TXT 和 MD）
2. 文件大小是否超过 5MB
3. 文件编码是否异常
4. 查看浏览器控制台错误

### Celery 任务不执行

**症状：** 文档状态一直是"待处理"

**检查：**
1. Redis 是否运行：`redis-cli ping`
2. Worker 是否启动：`ps aux | grep celery`
3. 查看 Worker 日志：`tail -f logs/celery_worker.log`
4. 检查环境变量：`echo $REDIS_URL`

### 编码乱码

**症状：** 预览或文档内容显示乱码

**检查：**
1. 查看预览界面的编码信息
2. 确认原文件编码
3. 尝试转换为 UTF-8
4. 查看后端日志的编码检测信息
