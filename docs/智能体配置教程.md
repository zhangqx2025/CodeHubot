## 🎯 智能体 1：猪猪侠对话

### 基本信息

| 字段 | 值 |
|------|-----|
| **智能体名称** | 猪猪侠对话 |
| **描述** | 一个友好的闲聊助手，可以和用户进行日常对话 |
| **大模型** | 通义千问-Turbo（或其他可用模型） |
| **状态** | 激活 |

### 系统提示词配置

```
你是一个友好、热情的对话助手，名字叫"猪猪侠"。你的特点是：

【性格特征】
• 活泼开朗，充满正能量
• 说话简洁明了，不啰嗦
• 善于倾听，理解用户情绪
• 偶尔会用一些可爱的表情符号 😊

【对话风格】
• 使用简体中文
• 语气轻松自然
• 回复长度适中（50-100字）
• 避免过于正式或生硬

【能力范围】
• 日常闲聊
• 情感支持
• 知识问答
• 讲笑话和故事

【示例对话】
用户："今天天气真好"
你："是呀！这么好的天气，最适合出去走走啦 ☀️ 你有什么户外计划吗？"

【注意】
- 当遇到需要专业技术支持的问题时，坦诚告知建议咨询专业人士
- 保持友善和尊重
```

### 插件配置

**不需要配置插件**（纯闲聊型智能体）

### 创建步骤

1. 进入"智能体管理"页面
2. 点击"创建智能体"
3. 填写基本信息：
   - 智能体名称：`猪猪侠对话`
   - 描述：`一个友好的闲聊助手`
4. 选择大模型：
   - 选择：`通义千问-Turbo`
   - Temperature：`0.7`（保持较高的创造性）
5. 粘贴上述系统提示词
6. 不配置插件
7. 状态选择：`激活`
8. 点击"保存"

## 🗑️ 智能体 2：垃圾分类助手

### 基本信息

| 字段 | 值 |
|------|-----|
| **智能体名称** | 垃圾分类助手 |
| **描述** | 智能垃圾分类，自动控制垃圾桶LED指示灯 |
| **大模型** | 通义千问-Turbo |
| **状态** | 激活 |

### 系统提示词配置

```
你是一个专业的垃圾分类助手，你必须严格执行以下操作流程：

【重要】每次用户提到要扔垃圾时，你必须：
1. 立即调用 executePreset 函数执行对应的预设指令
2. 等待函数执行完成
3. 然后向用户报告执行结果

【分类规则】你必须立即执行以下预设：

• 可回收垃圾（纸张、塑料瓶、金属罐、玻璃瓶、纸板箱等）
  → 立即执行预设：led_seq_mig4wqy5
  
• 有毒有害垃圾（电池、灯管、油漆、药品、杀虫剂、手机等电子产品）
  → 立即执行预设：led_seq_mig5hiep
  
• 厨余垃圾（剩饭剩菜、果皮、菜叶、茶叶渣、骨头、蛋壳等）
  → 立即执行预设：led_seq_mig5i461
  
• 其他垃圾（烟头、污染纸张、废旧陶瓷、一次性餐具、灰土等）
  → 立即执行预设：led_seq_mig5ivu5

【示例对话】
用户："我要扔报纸"
你的操作：
1. 调用 executePreset(device_uuid="2b91058a-xxx", preset_name="led_seq_mig4wqy5")
2. 回复："已打开可回收垃圾桶，LED灯已亮起3秒提示。报纸属于可回收垃圾。"

用户："手机坏了想扔掉"
你的操作：
1. 调用 executePreset(device_uuid="2b91058a-xxx", preset_name="led_seq_mig5hiep")
2. 回复："已打开有毒有害垃圾桶，LED灯已亮起3秒提示。手机属于电子垃圾，请投入有毒有害垃圾桶。"

【禁止】
- ❌ 不要只告诉用户要执行什么，必须真正执行
- ❌ 不要说"我将执行..."，而要先执行再说"已执行..."
- ❌ 不要跳过函数调用环节
```

### 插件配置

### 创建步骤

#### 第1步：创建预设指令

在创建智能体之前，需要先创建4个预设指令：

##### 1.1 可回收垃圾预设

```json
{
  "preset_key": "led_seq_mig4wqy5",
  "name": "可回收垃圾提示",
  "type": "sequence",
  "steps": [
    {
      "command": {
        "cmd": "led",
        "device_id": 1,
        "action": "on"
      },
      "delay": 3
    },
    {
      "command": {
        "cmd": "led",
        "device_id": 1,
        "action": "off"
      },
      "delay": 0
    }
  ]
}
```

**创建方式**：
1. 进入"设备管理" → 选择你的设备
2. 点击"设备配置" → "预设指令"
3. 点击"添加预设"
4. 填写预设名称：`可回收垃圾提示`
5. 预设类型：`序列指令`
6. 添加步骤1：LED1 开启，延迟3秒
7. 添加步骤2：LED1 关闭
8. 保存后复制生成的 `preset_key`（如：led_seq_mig4wqy5）

##### 1.2 有毒有害垃圾预设

## 🔧 大模型配置（重要）

### Temperature 参数说明

| 智能体类型 | 推荐 Temperature | 原因 |
|-----------|-----------------|------|
| **闲聊型**（猪猪侠） | 0.7 | 更有创造性，对话更自然 |
| **工具调用型**（垃圾分类） | 0.2 | 行为更确定，Function Calling 更稳定 |

### 修改 Temperature 的方法

#### 方法1：在前端修改（推荐）

1. 进入"大模型管理"
2. 找到"通义千问-Turbo"
3. 点击"编辑"
4. 修改 Temperature：`0.2`
5. 保存

#### 方法2：直接修改数据库

```sql
-- 查看当前配置
SELECT name, temperature FROM aiot_llm_models WHERE name = 'qwen-turbo';

-- 修改为 0.2
UPDATE aiot_llm_models 
SET temperature = 0.20 
WHERE name = 'qwen-turbo';
```

## 🐛 常见问题

### 问题1：垃圾分类助手有时候不调用函数

**原因**：Temperature 太高（0.7）

**解决方案**：
```sql
UPDATE aiot_llm_models 
SET temperature = 0.20 
WHERE name = 'qwen-turbo';
```

### 问题2：提示"无权使用此智能体"

**原因**：智能体不是你创建的

**解决方案**：
- 确保使用自己创建的智能体
- 或者联系管理员授权

### 问题3：预设指令执行失败（404）

**原因**：preset_key 不存在或拼写错误

**解决方案**：
1. 检查系统提示词中的 preset_key
2. 在设备配置中查看实际的 preset_key
3. 确保两者一致

### 问题4：LED 不亮

**检查清单**：
```bash
# 1. 检查设备是否在线
curl http://localhost:8000/api/devices/{device_uuid}

# 2. 检查 MQTT 是否正常
docker ps | grep mqtt

# 3. 检查 plugin-backend-service 日志
tail -f /opt/CodeHubot/nohup.out | grep preset
```

## 🎓 最佳实践

### 1. 系统提示词设计原则

✅ **明确的指令**
```
你必须... （而不是"你可以..."）
立即执行... （而不是"可以考虑执行..."）
```

✅ **具体的示例**
```
【示例对话】
用户："xxx"
你的操作：1. 调用函数  2. 回复内容
```

✅ **清晰的禁止项**
```
【禁止】
- ❌ 不要...
- ❌ 禁止...
```

### 2. Function Calling 稳定性

| 配置项 | 推荐值 | 原因 |
|--------|--------|------|
| Temperature | 0.1-0.3 | 降低随机性 |
| Top P | 0.9 | 保持适当多样性 |
| Max Tokens | 4096 | 足够的输出空间 |

### 3. 预设指令设计

**序列指令结构**：
```json
{
  "steps": [
    {
      "command": {...},
      "delay": 3  // 延迟时间（秒）
    }
  ]
}
```

**最佳延迟时间**：
- 提示灯：3秒
- 舵机动作：1-2秒
- 连续动作：0.5秒
