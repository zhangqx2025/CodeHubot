# 大文档向量化优化方案

## 🎯 问题描述

大文档向量化时常见的失败原因：
- ❌ 文本块数量过多，一次性API调用超时
- ❌ 网络波动导致部分请求失败
- ❌ API限流，短时间请求过多
- ❌ 内存不足，处理大量向量数据
- ❌ 数据库事务超时

---

## ✅ 解决方案

我已经实现了一套完整的优化方案：

### 1️⃣ 批量处理（Batch Processing）

**优化前**：
```python
# 一次性处理所有文本块
texts = [chunk['content'] for chunk in chunks_data]  # 可能有几百个
embeddings = await embedding_service.embed_texts(texts)  # 一次请求
```

**优化后**：
```python
# 分批处理，每批10个文本块
for batch_start in range(0, total_chunks, batch_size):
    batch_chunks = chunks_data[batch_start:batch_end]
    embeddings = await embedding_service.embed_texts(batch_texts)
    # 立即保存到数据库
    db.commit()
```

**效果**：
- ✅ 避免单次请求过大
- ✅ 降低超时风险
- ✅ 及时释放内存
- ✅ 失败只影响当前批次

### 2️⃣ 自动重试（Auto Retry）

```python
max_retries = 3
retry_delay = 2

for attempt in range(max_retries):
    try:
        embeddings = await embedding_service.embed_texts(texts)
        break  # 成功则退出重试
    except Exception as e:
        if attempt < max_retries - 1:
            await asyncio.sleep(retry_delay)
            retry_delay *= 2  # 指数退避
```

**效果**：
- ✅ 网络波动自动恢复
- ✅ 临时错误不影响整体
- ✅ 指数退避避免频繁重试

### 3️⃣ 进度保存（Progress Tracking）

```python
# 每个批次立即提交到数据库
for batch in batches:
    process_batch(batch)
    db.commit()  # 立即保存
    logger.info(f"批次 {i} 完成")
```

**效果**：
- ✅ 失败后已处理的数据不丢失
- ✅ 可以查看实时进度
- ✅ 支持断点续传

### 4️⃣ 限流保护（Rate Limiting）

```python
# 批次之间延迟
if batch_end < total_chunks:
    await asyncio.sleep(0.5)  # 延迟500ms
```

**效果**：
- ✅ 避免API限流
- ✅ 降低服务器压力
- ✅ 提高成功率

### 5️⃣ 详细日志（Detailed Logging）

```python
logger.info(f"处理批次 {batch_num}/{total_batches} ({start}-{end}/{total})")
logger.warning(f"批次失败，重试中...")
logger.error(f"详细错误: {traceback.format_exc()}")
```

**效果**：
- ✅ 实时了解处理进度
- ✅ 快速定位问题
- ✅ 便于故障排查

## 🔧 配置参数

### 批量大小（batch_size）

```python
# 默认值：10
batch_size = 10  # 每批处理10个文本块
```

**调整建议**：
- 🟢 **网络稳定** → 增大到 20-30
- 🟡 **网络一般** → 保持默认 10
- 🔴 **网络不稳** → 减小到 5

### 重试次数（max_retries）

```python
# 默认值：3
max_retries = 3  # 最多重试3次
```

**调整建议**：
- 重要文档 → 增加到 5
- 普通文档 → 保持默认 3
- 测试文档 → 减少到 1

### 重试延迟（retry_delay）

```python
# 默认值：2秒，指数增长
retry_delay = 2  # 首次重试延迟2秒，后续4秒、8秒...
```

## 💡 使用建议

### 场景 1: 超大文档（> 500个文本块）

**问题**：向量化耗时过长

**解决方案**：
1. **增大文本块大小**
   - 上传时选择"自定义"
   - 设置块大小为 800-1000 字符
   - 减少文本块总数

2. **调整切分方式**
   - 使用"按段落切分"
   - 段落自然合并，减少块数

3. **分割文档**
   - 将大文档拆分成多个小文档
   - 分别上传和向量化

### 场景 2: 网络不稳定

**问题**：频繁超时失败

**解决方案**：
1. **减小批量大小**
   ```python
   # 在代码中调整
   batch_size = 5  # 从10降到5
   ```

2. **增加重试次数**
   ```python
   max_retries = 5  # 从3增加到5
   ```

3. **使用断点续传**
   - 失败后重新触发向量化
   - 已成功的批次不会重复处理

### 场景 3: API限流

**问题**：提示"超出配额"或"请求过于频繁"

**解决方案**：
1. **增加批次延迟**
   ```python
   await asyncio.sleep(1.0)  # 从0.5秒增加到1秒
   ```

2. **充值API额度**
   - 访问阿里云控制台充值

3. **分时段处理**
   - 避开高峰时段
   - 在夜间处理大文档

import asyncio
from app.core.database import SessionLocal
from app.services.embedding_service import embed_document

async def process_with_custom_params(doc_id):
    db = SessionLocal()
    try:
        await embed_document(

```sql
-- 查看已处理的文本块数量

    d.id,
    d.title,
    d.chunk_count as expected_chunks,
    COUNT(dc.id) as actual_chunks,
    d.embedding_status
FROM aiot_documents d
LEFT JOIN aiot_document_chunks dc ON d.id = dc.document_id
WHERE d.embedding_status = 'failed'
GROUP BY d.id
HAVING actual_chunks > 0;
```

## 🔄 断点续传

### 原理

优化后的向量化函数会：
1. 按批次处理文本块
2. 每批次立即提交到数据库
3. 失败时，已成功的批次保留

### 使用方法

```bash
# 1. 第一次向量化（可能部分失败）
python scripts/manual_embed_document.py 123

# 2. 查看已处理的数量
python -c "
from app.core.database import SessionLocal
from app.models.document import DocumentChunk
db = SessionLocal()
count = db.query(DocumentChunk).filter(DocumentChunk.document_id == 123).count()
print(f'已处理: {count} 个文本块')
"

# 3. 重新触发（会跳过已处理的部分）
# 注意：当前版本会重新处理，未来版本会支持真正的断点续传
```

## ⚠️ 注意事项

### API限制

阿里云通义千问API限制：
- 每秒请求数（QPS）：根据账户等级
- 每分钟Token数：根据账户等级
- 单次请求文本数：建议不超过25个

**建议**：
- 批量大小设置为 10-15
- 批次间延迟至少 0.5 秒
- 高峰时段避免大批量处理

### 内存管理

```python
# 处理完大批量后，释放内存
import gc
gc.collect()
```

### 数据库连接

```python
# 长时间处理时，刷新数据库连接
if time.time() - last_refresh > 300:  # 5分钟
    db.close()
    db = SessionLocal()
    last_refresh = time.time()
```

## 📞 问题排查

### 问题 1: 一直处理中

**症状**：文档状态长时间显示"处理中"

**排查**：
```bash
# 查看后端日志
tail -100 logs/app.log | grep "文档 123"

# 检查进程状态
ps aux | grep python | grep embed
```

**解决**：
- 如果进程存在，耐心等待
- 如果进程不存在，重新触发

### 问题 2: 部分失败

**症状**：有些批次成功，有些失败

**排查**：
```sql
SELECT embedding_error FROM aiot_documents WHERE id = 123;
```

**常见错误**：
- "timeout" → 增加重试次数
- "rate limit" → 增加批次延迟
- "invalid content" → 检查文本内容

### 问题 3: 内存不足

**症状**：服务器内存占用过高

**排查**：
```bash
# 查看内存使用
free -h
top -p $(pgrep -f uvicorn)
```

**解决**：
- 减小批量大小
- 增加批次延迟
- 重启服务释放内存
