# 个人信息管理功能说明

## 📋 功能概述

本次更新实现了完整的个人信息管理功能，包括：
1. **修改密码**：用户可以修改自己的登录密码
2. **修改个人信息**：用户可以修改用户名和邮箱

## 🔧 后端实现

### 1. Schema定义

**位置**: `backend/app/schemas/user.py`

#### ChangePasswordRequest
```python
class ChangePasswordRequest(BaseModel):
    """修改密码请求Schema"""
    old_password: str  # 当前密码
    new_password: str  # 新密码（8位以上，包含字母和数字）
```

**验证规则**:
- 新密码不能与旧密码相同
- 最小长度8位
- 必须包含字母和数字
- 不能包含空格
- 不能是常见弱密码

#### UpdateProfileRequest
```python
class UpdateProfileRequest(BaseModel):
    """修改个人信息请求Schema"""
    email: Optional[EmailStr]     # 邮箱（可选）
    username: Optional[str]       # 用户名（可选）
```

**验证规则**:
- 用户名：3-50个字符，仅支持字母、数字、下划线、连字符
- 邮箱：标准邮箱格式验证
- 不能使用已被占用的用户名或邮箱

### 2. API接口

**位置**: `backend/app/api/auth.py`

#### POST /api/auth/change-password
修改密码接口

**请求体**:
```json
{
  "old_password": "OldPass123",
  "new_password": "NewPass456"
}
```

**响应**:
```json
{
  "message": "密码修改成功"
}
```

**错误码**:
- `400`: 当前密码错误
- `400`: 新密码验证失败
- `500`: 服务器错误

#### PUT /api/auth/profile
修改个人信息接口

**请求体**:
```json
{
  "username": "new_username",
  "email": "new_email@example.com"
}
```

**响应**:
```json
{
  "id": 1,
  "username": "new_username",
  "email": "new_email@example.com",
  "role": "user",
  "is_active": true,
  "created_at": "2025-11-27T00:00:00",
  "updated_at": "2025-11-27T10:30:00"
}
```

**错误码**:
- `400`: 没有提供更新内容
- `400`: 用户名已被使用
- `400`: 邮箱已被使用
- `500`: 服务器错误

## 🎨 前端实现

### 1. API调用

**位置**: `frontend/src/api/auth.js`

```javascript
// 修改密码
export const changePassword = (data) => {
  return request.post('/auth/change-password', data)
}

// 修改个人信息
export const updateProfile = (data) => {
  return request.put('/auth/profile', data)
}
```

### 2. 用户界面

**位置**: `frontend/src/views/Profile.vue`

#### 个人信息卡片
- 显示用户头像、用户名、邮箱

- 显示设备统计信息（设备数量、在线设备、登录天数）

#### 修改个人信息表单

**表单验证**:
- 当前密码：必填
- 新密码：必填，最少8位，必须包含字母和数字
- 确认密码：必填，必须与新密码一致

## 📊 数据库更新

本功能使用现有的 `aiot_core_users` 表，无需额外的数据库迁移：

```sql
-- 查看用户信息
SELECT id, username, email, created_at, updated_at 
FROM aiot_core_users 
WHERE id = 1;

-- 验证密码是否已更新（password_hash会变化）
SELECT id, username, password_hash, updated_at 
FROM aiot_core_users 
WHERE id = 1;
```

## ⚠️ 注意事项

1. **密码安全**：
   - 所有密码都使用bcrypt加密存储
   - 修改密码必须验证旧密码
   - 密码要求：最少8位，必须包含字母和数字

2. **唯一性验证**：
   - 用户名在全系统内必须唯一
   - 邮箱在全系统内必须唯一
   - 修改时会自动检查是否冲突

3. **权限控制**：
   - 所有接口都需要JWT token认证
   - 用户只能修改自己的信息
   - Token过期后需要重新登录

4. **前端验证**：
   - 前端有基本的表单验证
   - 后端有完整的数据验证
   - 两层验证确保数据安全

5. **错误处理**：
   - 所有错误都有清晰的提示信息
   - 敏感错误不会暴露系统细节
   - 日志记录所有操作便于排查问题
