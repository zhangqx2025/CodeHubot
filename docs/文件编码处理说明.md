# 文件编码处理说明

本文档说明知识库系统如何处理文件编码问题，以及如何避免和解决编码相关的错误。

## 📖 什么是文件编码？

文件编码是将文本字符转换为计算机可以存储的二进制数据的方式。不同的编码方式会导致同一文本产生不同的二进制表示。

### 常见编码格式

| 编码格式 | 说明 | 使用场景 |
|---------|------|---------|
| **UTF-8** | 国际通用编码，支持所有语言 | ✅ **推荐使用**<br>跨平台、跨语言最佳选择 |
| **GBK** | 中文编码，支持简繁体中文 | Windows 中文系统默认编码 |
| **GB2312** | 简体中文编码（GBK 的子集） | 早期中文系统 |
| **GB18030** | 中文国标编码（最完整） | 政府、金融等特定领域 |
| **BIG5** | 繁体中文编码 | 台湾、香港地区 |
| **Latin-1** | 西欧语言编码 | 欧洲语言文档 |

## 🎯 系统的编码处理策略

### 智能编码检测

系统会自动检测上传文件的编码格式：

1. **使用 chardet 库检测**
   - 分析文件内容特征
   - 给出编码类型和置信度

2. **多编码尝试**
   - 如果检测置信度低（< 70%），尝试常见编码
   - 依次尝试：UTF-8 → GBK → GB2312 → GB18030 → BIG5

3. **智能回退**
   - 如果所有编码都失败，使用 UTF-8 并替换无法识别的字符
   - 检测乱码比例，超过 10% 则报错提示用户

### 编码检测示例

```python
# 系统会在日志中显示检测结果
INFO: 检测到文件编码: utf-8 (置信度: 99.00%)
INFO: 成功使用 utf-8 编码解析文件
```

或者

```python
INFO: 检测到文件编码: gbk (置信度: 85.50%)
INFO: 成功使用 gbk 编码解析文件
```

## ⚠️ 常见编码问题

### 问题1：乱码

**症状：**
- 上传文件后预览显示乱码
- 中文变成 "锟斤拷"、"烫烫烫" 等
- 出现大量 "?" 或方框字符

**原因：**
- 文件使用非 UTF-8 编码保存
- 系统编码检测失败
- 文件本身已损坏

**解决方法：**
1. 将文件转换为 UTF-8 编码（见下方转换方法）
2. 使用"预览切分"功能，查看系统检测到的编码
3. 如果编码正确但仍有乱码，文件可能已损坏，需重新创建

### 问题2：上传失败（编码错误）

**症状：**
```
文件编码错误，包含 15.2% 的乱码字符。
请确保文件使用 UTF-8、GBK 或 GB2312 编码
```

**原因：**
- 文件编码无法识别
- 包含大量无效字符

**解决方法：**
1. 使用专业编辑器（如 VS Code、Notepad++）转换编码
2. 检查文件是否包含特殊字符或控制字符
3. 尝试重新创建文件

### 问题3：部分字符显示异常

**症状：**
- 大部分内容正常，但某些字符显示为 "�"
- 特殊符号丢失

**原因：**
- 原文件混用了多种编码
- 特殊字符在当前编码中不存在

**解决方法：**
1. 统一使用 UTF-8 编码
2. 检查并清理特殊字符
3. 使用"另存为"并选择 UTF-8 编码

## 🔧 如何转换文件编码

### 方法1：使用 VS Code（推荐）

1. 用 VS Code 打开文件
2. 点击右下角的编码名称（如 "GBK"）
3. 选择"通过编码重新打开"
4. 选择原编码（如 GBK）打开文件
5. 再次点击右下角编码
6. 选择"通过编码保存"
7. 选择 "UTF-8"
8. 保存文件

**详细步骤：**
```
打开文件 → 右下角编码 → 重新打开 → 选择原编码（GBK/GB2312）
→ 右下角编码 → 通过编码保存 → UTF-8 → 保存
```

### 方法2：使用 Notepad++

1. 用 Notepad++ 打开文件
2. 菜单：编码 → 转换为 UTF-8（带 BOM）或 UTF-8
3. 保存文件

**推荐：**
- 对于纯文本文件：UTF-8（不带 BOM）
- 对于某些 Windows 程序：UTF-8 带 BOM

### 方法3：使用 Python 脚本（批量转换）

创建 `convert_to_utf8.py`：

```python
#!/usr/bin/env python3
import sys
import chardet

def convert_file_to_utf8(file_path):
    # 读取文件
    with open(file_path, 'rb') as f:
        content = f.read()
    
    # 检测编码
    detected = chardet.detect(content)
    encoding = detected['encoding']
    print(f"检测到编码: {encoding} (置信度: {detected['confidence']:.2%})")
    
    # 解码
    try:
        text = content.decode(encoding)
    except:
        print("尝试使用 GBK 编码...")
        text = content.decode('gbk', errors='ignore')
    
    # 保存为 UTF-8
    output_path = file_path + '.utf8.txt'
    with open(output_path, 'w', encoding='utf-8') as f:
        f.write(text)
    
    print(f"已转换并保存到: {output_path}")

if __name__ == '__main__':
    if len(sys.argv) < 2:
        print("用法: python convert_to_utf8.py <文件路径>")
        sys.exit(1)
    
    convert_file_to_utf8(sys.argv[1])
```

**使用：**
```bash
python convert_to_utf8.py your_file.txt
```

### 方法4：使用 Linux/macOS 命令行

**使用 iconv：**
```bash
# 从 GBK 转换为 UTF-8
iconv -f GBK -t UTF-8 input.txt -o output.txt

# 从 GB2312 转换为 UTF-8
iconv -f GB2312 -t UTF-8 input.txt -o output.txt
```

**批量转换：**
```bash
# 转换目录下所有 .txt 文件
for file in *.txt; do
    iconv -f GBK -t UTF-8 "$file" -o "${file}.utf8"
done
```

### 方法5：使用 Windows 记事本

1. 用记事本打开文件
2. 文件 → 另存为
3. 编码下拉框选择 "UTF-8"
4. 保存

⚠️ **注意：** Windows 记事本可能添加 BOM 标记，系统会自动处理，但建议使用专业编辑器。

## ✅ 最佳实践

### 创建新文档时

1. **始终使用 UTF-8 编码保存**
   - 跨平台兼容性最好
   - 支持所有语言字符
   - 不会产生乱码问题

2. **使用专业编辑器**
   - 推荐：VS Code、Sublime Text、Notepad++
   - 避免：Windows 记事本（可能添加 BOM）

3. **检查文件编码**
   - 保存后用编辑器确认编码格式
   - 或使用 `file` 命令（Linux/macOS）：
     ```bash
     file -i your_file.txt
     # 输出: your_file.txt: text/plain; charset=utf-8
     ```

### 处理已有文档时

1. **先转换编码**
   - 批量转换所有文档为 UTF-8
   - 保留原文件备份

2. **使用预览功能**
   - 上传前先预览，确认编码正确
   - 检查是否有乱码

3. **分批上传**
   - 不确定的文档先小批量测试
   - 确认无误后再批量上传

## 🔍 故障排查

### 检查文件编码

**Python 方法：**
```python
import chardet

with open('your_file.txt', 'rb') as f:
    result = chardet.detect(f.read())
    print(result)
# 输出: {'encoding': 'utf-8', 'confidence': 0.99, 'language': ''}
```

**Linux/macOS 方法：**
```bash
file -i your_file.txt
# 输出: text/plain; charset=utf-8

# 或使用 enca
enca -L zh_CN your_file.txt
# 输出: Simplified Chinese National Standard; GB2312
```

**Windows 方法：**
- 使用 Notepad++ 打开，查看右下角编码显示
- 或使用 VS Code 打开，查看右下角编码

### 验证转换结果

转换后，再次检查：


## 📚 技术细节

### 系统编码检测流程

```
1. 读取文件二进制内容
   ↓
2. 使用 chardet 检测编码
   ↓
3. 置信度 ≥ 70%？
   ├─ 是 → 使用检测到的编码
   └─ 否 → 尝试常见编码列表
       ↓
4. 依次尝试：UTF-8 → GBK → GB2312 → GB18030 → BIG5
   ↓
5. 成功解码？
   ├─ 是 → 继续处理
   └─ 否 → 使用 UTF-8 + replace 错误处理
       ↓
6. 检查乱码比例
   ↓
7. 乱码 > 10%？
   ├─ 是 → 抛出错误，要求用户转换编码
   └─ 否 → 继续处理文档
```

### 编码检测算法

系统使用 `chardet` 库，基于：
- 字符频率分析
- 字符组合模式
- 多字节序列特征

**准确率：**
- UTF-8：> 99%
- GBK/GB2312：85-95%
- 其他编码：70-90%

### BOM（字节顺序标记）处理

系统自动处理 BOM：
- UTF-8 with BOM (`\ufeff`)：自动去除
- UTF-16/UTF-32 BOM：暂不支持（建议转换为 UTF-8）

## ❓ 常见问题

**Q: 为什么推荐使用 UTF-8？**

A: UTF-8 是国际标准，具有以下优势：
- 兼容 ASCII
- 支持所有 Unicode 字符（140+ 语言）
- 跨平台兼容性最好
- 文件大小适中（中文约 3 字节/字符）

**Q: GBK 编码有什么问题吗？**

A: GBK 本身没问题，但：
- 仅支持中文，不适合国际化
- 在非中文系统上可能出现乱码
- 与某些工具兼容性差

建议：如果文件只在中文环境使用，GBK 可以接受；但为了长远考虑，最好转换为 UTF-8。

**Q: 系统会修改我的原文件吗？**

A: 不会。系统只是读取文件内容，不会修改原文件。向量化处理的是解析后的文本，原文件保持不变。

**Q: 如果编码检测错误怎么办？**

A: 
1. 使用"预览切分"功能，检查是否有乱码
2. 如果有乱码，说明编码检测失败
3. 手动将文件转换为 UTF-8 后重新上传

**Q: 支持其他语言（英文、日文等）吗？**

A: 完全支持。UTF-8 支持所有 Unicode 字符，包括：
- 英文、法文、德文等拉丁字符
- 日文（平假名、片假名、汉字）
- 韩文（谚文）
- 阿拉伯文、西里尔文等

**Q: 二进制文件（PDF、Word）如何处理？**

A: 目前系统只支持纯文本（TXT）和 Markdown（MD）格式。对于其他格式：
- PDF：需要先转换为 TXT
- Word：需要先转换为 TXT 或 MD
- 未来可能支持更多格式
