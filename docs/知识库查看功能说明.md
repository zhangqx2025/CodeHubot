# 知识库向量化查看功能说明

## 📋 功能概述

已实现完整的文档向量化查看功能，用户可以查看：
- ✅ 文档基本信息
- ✅ 向量化状态和统计
- ✅ 文本块列表
- ✅ 每个文本块的详细内容
- ✅ 向量化状态标识

## 🎯 使用方式

### 方式 1: 前端界面查看（推荐）

#### 步骤 1: 进入知识库详情
1. 登录系统
2. 左侧菜单 → **AI 智能体** → **知识库管理**
3. 点击某个知识库进入详情页

#### 步骤 2: 查看文档详情
1. 在文档列表中找到已向量化的文档（状态显示为"✅ 已完成"）
2. 点击该文档右侧的 **"查看"** 按钮
3. 弹出文档详情对话框

#### 步骤 3: 浏览文本块
在文档详情对话框中，您可以看到：

**文档基本信息**：
- 文件类型（TXT/MD）
- 文件大小
- 向量化状态
- 文本块数量
- 上传时间
- 向量化完成时间

**文本块列表**（表格形式）：
- 序号：文本块的索引
- 内容预览：显示前两行内容
- 字符数：该文本块的字符数量
- Token数：估算的 Token 数量
- 向量状态：✅ 已向量化 / ❌ 无向量
- 操作：点击"查看详情"可以看到完整内容

#### 步骤 4: 查看文本块详情
1. 点击某个文本块的 **"查看详情"** 按钮
2. 弹出对话框显示该文本块的完整内容
3. 可以复制内容用于测试或其他用途

### 方式 2: 数据库查询

如果需要更详细的数据或批量查询，可以直接查询数据库：

#### 查询文档向量化状态
```sql
SELECT 
    d.id,
    d.title,
    d.embedding_status,
    d.chunk_count,
    d.embedded_at,
    kb.name as knowledge_base_name
FROM aiot_documents d
LEFT JOIN aiot_knowledge_bases kb ON d.knowledge_base_id = kb.id
WHERE d.embedding_status = 'completed'
  AND d.deleted_at IS NULL
ORDER BY d.embedded_at DESC;
```

#### 查询文档的所有文本块
```sql
SELECT 
    id,
    chunk_index,
    LEFT(content, 100) as content_preview,
    char_count,
    token_count,
    CASE 
        WHEN embedding_vector IS NOT NULL THEN '✅ 有向量'
        ELSE '❌ 无向量'
    END as has_vector
FROM aiot_document_chunks
WHERE document_id = 1  -- 替换为您的文档ID
ORDER BY chunk_index;
```

#### 查看向量维度
```sql
SELECT 
    id,
    chunk_index,
    JSON_LENGTH(embedding_vector) as vector_dimension
FROM aiot_document_chunks
WHERE document_id = 1
  AND embedding_vector IS NOT NULL
LIMIT 5;
```

## 📊 界面功能说明

### 文档列表

| 列名 | 说明 |
|------|------|
| 标题 | 文档名称 |
| 类型 | TXT 或 MD |
| 大小 | 文件大小（自动格式化） |
| 向量化状态 | 待处理/处理中/已完成/失败 |
| 文本块 | 切分后的文本块数量 |
| 上传时间 | 文档上传时间 |
| 操作 | 查看、向量化、删除 |

### 向量化状态标识

| 状态 | 标签颜色 | 说明 | 可用操作 |
|------|---------|------|---------|
| `pending` | 灰色 | 待处理 | 可触发向量化 |
| `processing` | 蓝色 | 处理中 | 请等待 |
| `completed` | 绿色 | 已完成 | 可查看详情 |
| `failed` | 红色 | 失败 | 可重新向量化 |

### 文本块列表

**列说明**：
- **序号**：文本块在文档中的顺序（从0开始）
- **内容预览**：显示文本块的前两行，支持换行显示
- **字符数**：该文本块包含的字符数量
- **Token数**：估算的 Token 数量（用于计算成本）
- **向量状态**：
  - ✅ 已向量化：该文本块已成功生成向量
  - ❌ 无向量：向量化失败或未完成
- **查看详情**：点击后显示该文本块的完整内容

## 💡 使用技巧

### 技巧 1: 快速检查向量化结果
1. 在文档列表中，**文本块**列显示了切分后的数量
2. 如果数量为 0，说明还未向量化
3. 点击"查看"可以验证向量化质量

### 技巧 2: 验证文本切分效果
1. 打开文档详情
2. 查看文本块列表，检查：
   - 切分是否合理
   - 每个块的字符数是否适中（建议 300-600 字符）
   - 内容是否完整（没有截断句子）

### 技巧 3: 排查向量化问题
如果向量化后搜索效果不好：
1. 查看文本块列表，检查是否所有块都有向量
2. 查看文本块内容，确认切分质量
3. 检查 Token 数量是否合理

### 技巧 4: 批量查看
- 使用 Python 脚本可以快速查看所有文档的向量化状态
- 使用 SQL 查询可以批量统计和分析

## ❓ 常见问题

### Q1: 为什么有些文本块显示"❌ 无向量"？

**A**: 可能的原因：
1. 向量化过程中出现错误
2. 文本块内容为空或过短
3. API 调用失败

**解决方法**：
- 查看文档的 `embedding_error` 字段
- 重新触发向量化
- 检查 API 配额和余额

### Q2: 如何判断文本切分质量？

**A**: 好的切分应该：
- ✅ 每个块 300-600 字符
- ✅ 不截断句子（在句号、换行处切分）
- ✅ 保留语义完整性
- ✅ 避免过长或过短的块

可以在文档详情中查看并评估。

### Q3: 向量维度是什么意思？

**A**: 
- 向量维度：向量数组的长度
- 阿里云通义千问：1536 维
- OpenAI text-embedding-ada-002：1536 维
- 维度越高，表达能力越强，但存储成本也越高

### Q4: 如何导出文本块数据？

**A**: 使用 SQL 导出：
```sql
SELECT 
    d.title as document_title,
    dc.chunk_index,
    dc.content,
    dc.char_count,
    dc.token_count
FROM aiot_document_chunks dc
JOIN aiot_documents d ON dc.document_id = d.id
WHERE d.knowledge_base_id = 1  -- 替换为您的知识库ID
ORDER BY d.id, dc.chunk_index
INTO OUTFILE '/tmp/chunks_export.csv'
FIELDS TERMINATED BY ',' 
ENCLOSED BY '"'
LINES TERMINATED BY '\n';
```
