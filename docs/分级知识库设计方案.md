## 系统概述

### 设计目标

构建一个**极简但强大**的分级知识库系统，支持：
- ✅ 多层级知识组织（系统/学校/课程/个人）
- ✅ 智能体与知识库灵活关联
- ✅ 向量检索与语义搜索
- ✅ 权限自动管理（零配置）
- ✅ 文档管理与向量化

### 技术栈

| 组件 | 技术选型 | 说明 |
|-----|---------|------|
| 向量存储 | MySQL (JSON字段) | 简化部署，无需额外服务 |
| 嵌入服务 | 阿里云通义千问 | 高质量中文支持 |
| 文件存储 | 本地文件系统 | 简单可靠 |
| 文档格式 | Markdown (.md), 纯文本 (.txt) | 轻量级，易于处理 |
| 异步处理 | 同步处理 | 简化架构 |

## 核心设计理念

### 1. 极简原则

> **"能自动的就不让用户选，能简化的就不做复杂"**

**实施成果**：
- ❌ 移除复杂的访问权限选择（3选1 → 自动）
- ❌ 移除专业的向量参数配置（5个 → 0个）
- ❌ 移除复杂的权限管理功能
- ✅ 用户只需选择知识库类型，系统自动处理其他

**效果对比**：

| 版本 | 必填字段 | 必选项 | 决策时间 | 上手难度 |
|-----|---------|-------|---------|---------|
| 初始版本 | 7个 | 5个 | ~2分钟 | ★★★★★ |
| 第一次简化 | 4个 | 3个 | ~1分钟 | ★★★☆☆ |
| **终极简化** | **3个** | **1个** | **~30秒** | **★☆☆☆☆** |

### 2. 智能关联

知识库通过**关联表**实现智能体使用，而不是"拥有"：

```
知识库池:
├─ 系统知识库（平台手册）
├─ 学校知识库（学校规章）
├─ 课程知识库（课程资料）
└─ 个人知识库（个人笔记）
         ↕️ 多对多关联
    智能体列表
```

**优势**：
- ✅ 一个知识库可被多个智能体使用
- ✅ 一个智能体可使用多个知识库
- ✅ 每个关联可独立配置优先级
- ✅ 知识库更新，所有智能体自动生效

## 知识库类型

### 类型定义（4种）

| 类型 | scope_type | scope_id | 可见范围 | 创建权限 |
|-----|-----------|----------|---------|---------|
| **系统级** | `system` | `NULL` | 所有人 | platform_admin |
| **学校级** | `school` | `school_id` | 本校师生 | platform_admin, school_admin |
| **课程级** | `course` | `course_id` | 课程师生 | 管理员、教师 |
| **个人级** | `personal` | `user_id` | 仅创建者 | 所有用户 |

**移除**: ~~智能体级 (agent)~~ - 通过关联表实现，不需要独立类型

### 可见性规则（自动）

系统根据 `scope_type` 自动设置访问权限，用户**无需选择**：

```javascript
const accessLevelMap = {
  'system': 'public',    // 系统级 → 所有人可见
  'school': 'protected', // 学校级 → 本校可见
  'course': 'protected', // 课程级 → 课程可见
  'personal': 'private'  // 个人级 → 仅自己可见
}
```

## 数据库设计

### 核心表结构

#### 1. 知识库表 (aiot_knowledge_bases)

```sql
CREATE TABLE `aiot_knowledge_bases` (
  `id` INT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
  `uuid` VARCHAR(36) NOT NULL UNIQUE COMMENT '知识库UUID',
  `name` VARCHAR(100) NOT NULL COMMENT '知识库名称',
  `description` TEXT COMMENT '知识库描述',
  `icon` VARCHAR(200) COMMENT '知识库图标URL',
  
  -- 层级与归属
  `scope_type` ENUM('system', 'school', 'course', 'personal') NOT NULL COMMENT '作用域类型',
  `scope_id` INT NULL COMMENT '作用域ID（school_id/course_id/user_id）',
  `parent_kb_id` INT NULL COMMENT '父知识库ID（支持继承）',
  
  -- 创建者与权限
  `owner_id` INT NOT NULL COMMENT '创建者用户ID',
  `access_level` ENUM('public', 'protected', 'private') DEFAULT 'private' COMMENT '访问级别',
  
  -- 统计信息
  `document_count` INT DEFAULT 0 COMMENT '文档数量',
  `chunk_count` INT DEFAULT 0 COMMENT '文本块数量',
  `total_size` BIGINT DEFAULT 0 COMMENT '总大小（字节）',
  `last_updated_at` DATETIME COMMENT '最后更新时间',
  
  -- 配置参数
  `chunk_size` INT DEFAULT 500 COMMENT '文本块大小',
  `chunk_overlap` INT DEFAULT 50 COMMENT '文本块重叠大小',
  `embedding_model_id` INT COMMENT 'Embedding模型ID',
  `retrieval_config` JSON COMMENT '检索配置（JSON格式）',
  
  -- 状态
  `is_active` TINYINT(1) DEFAULT 1 COMMENT '是否激活',
  `is_system` TINYINT(1) DEFAULT 0 COMMENT '是否系统知识库',
  
  -- 扩展字段
  `tags` JSON COMMENT '标签数组（JSON格式）',
  `meta_data` JSON COMMENT '扩展元数据（JSON格式）',
  
  -- 时间戳
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted_at` DATETIME NULL COMMENT '删除时间（软删除）',
  
  KEY `idx_scope` (`scope_type`, `scope_id`),
  KEY `idx_owner` (`owner_id`),
  KEY `idx_parent` (`parent_kb_id`),
  KEY `idx_deleted` (`deleted_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='知识库主表';
```

**作用域说明**：
- `system`: scope_id = NULL, 平台级知识库
- `school`: scope_id = school_id, 学校级知识库
- `course`: scope_id = course_id, 课程级知识库
- `personal`: scope_id = user_id, 个人知识库

## 权限与访问控制

### 权限设计原则

1. **自动化优先**：根据 scope_type 自动设置访问权限
2. **最小权限**：默认最严格，逐步开放
3. **清晰规则**：简单易懂的权限逻辑
4. **无需配置**：用户无需手动管理权限

### 访问权限矩阵

#### 系统级知识库 (system + public)

| 用户角色 | 读 | 写 | 删除 | 说明 |
|---------|----|----|------|------|
| 平台管理员 | ✅ | ✅ | ✅ | 完全控制 |
| 学校管理员 | ✅ | ❌ | ❌ | 只读访问 |
| 教师 | ✅ | ❌ | ❌ | 只读访问 |
| 学生 | ✅ | ❌ | ❌ | 只读访问 |

#### 学校级知识库 (school + protected)

| 用户角色 | 读 | 写 | 删除 | 说明 |
|---------|----|----|------|------|
| 平台管理员 | ✅ | ✅ | ✅ | 完全控制 |
| 本校管理员 | ✅ | ✅ | ✅ | 完全控制 |
| 本校教师 | ✅ | ❌ | ❌ | 只读访问 |
| 本校学生 | ✅ | ❌ | ❌ | 只读访问 |
| 其他学校用户 | ❌ | ❌ | ❌ | 无权访问 |

#### 课程级知识库 (course + protected)

| 用户角色 | 读 | 写 | 删除 | 说明 |
|---------|----|----|------|------|
| 平台管理员 | ✅ | ✅ | ✅ | 完全控制 |
| 创建者 | ✅ | ✅ | ✅ | 完全控制 |
| 任课教师 | ✅ | ✅ | ❌ | 可编辑 |
| 选课学生 | ✅ | ❌ | ❌ | 只读访问 |
| 其他用户 | ❌ | ❌ | ❌ | 无权访问 |

#### 个人级知识库 (personal + private)

| 用户角色 | 读 | 写 | 删除 | 说明 |
|---------|----|----|------|------|
| 创建者本人 | ✅ | ✅ | ✅ | 完全控制 |
| 平台管理员 | ❌ | ❌ | ❌ | **无权访问**⚠️ |
| 其他用户 | ❌ | ❌ | ❌ | 无权访问 |

**特别说明**：个人知识库是真正的"私有"，即使平台管理员也无法访问，保护用户隐私。

### 权限检查流程

```python
def check_kb_permission(user, kb, permission):
    # 1. 平台管理员（对非private知识库有完全权限）
    if user.role == 'platform_admin' and kb.access_level != 'private':
        return True
    
    # 2. 创建者
    if kb.owner_id == user.id:
        return True
    
    # 3. 个人知识库特殊处理
    if kb.scope_type == 'personal':
        return kb.scope_id == user.id
    
    # 4. 学校级权限
    if kb.scope_type == 'school':
        return user.school_id == kb.scope_id
    
    # 5. 课程级权限
    if kb.scope_type == 'course':
        # 检查是否是任课教师或选课学生
        return is_course_member(user, kb.scope_id)
    
    # 6. 系统级公开知识库
    if kb.scope_type == 'system' and kb.access_level == 'public':
        return permission == 'read'
    
    return False
```

## 智能体知识库关联

### 设计理念

> **智能体通过"关联"使用知识库，而不是"拥有"专属知识库**

### 关联配置

每个关联支持以下配置：

| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| `priority` | int (1-10) | 5 | 优先级，数字越大越优先 |
| `is_enabled` | boolean | true | 是否启用 |
| `top_k` | int | 5 | 返回文档数量 |
| `similarity_threshold` | float | 0.70 | 相似度阈值 |
| `retrieval_mode` | enum | hybrid | 检索模式 |

**简化方案**：用户只需配置 `priority` 和 `is_enabled`，其他使用默认值。

### 使用流程

```
1. 创建知识库
   └─ 上传文档 → 向量化

2. 创建智能体
   └─ 配置基本信息

3. 关联知识库
   ├─ 选择要使用的知识库（可多选）
   ├─ 系统自动分配优先级（10, 9, 8...）
   └─ 保存关联

4. 智能体工作
   └─ 按优先级检索知识库
       └─ 返回相关内容
```

### 实际案例

#### 案例：创建"数学助教"智能体

**步骤1：准备知识库**
```
✅ 系统知识库：平台使用教程
✅ 学校知识库：某某中学规章制度
✅ 课程知识库：高一数学教材
✅ 个人知识库：我的数学教学笔记
```

**步骤2：创建智能体**
```
名称：数学助教
提示词：你是一个专业的数学老师助手...
```

**步骤3：关联知识库**
```
优先级排序：
10 ━ 高一数学教材 [课程级] ✅ 启用
 8 ━ 我的数学教学笔记 [个人级] ✅ 启用
 5 ━ 某某中学规章制度 [学校级] ✅ 启用
 1 ━ 平台使用教程 [系统级] ✅ 启用
```

**步骤4：学生提问**
```
学生："勾股定理是什么？"

检索流程：
1️⃣ 高一数学教材(p=10) → 找到"勾股定理定义"
2️⃣ 我的教学笔记(p=8) → 找到"教学技巧"
3️⃣ 组合答案 → 回复学生

答案包含：
- 勾股定理定义（来自教材）
- 教学案例（来自笔记）
- 清晰易懂（智能体整合）
```

## API设计

### RESTful API 端点

#### 知识库管理

```http
# 创建知识库
POST /api/knowledge-bases
Body: {
  "name": "我的数学笔记",
  "description": "日常教学总结",
  "scope_type": "personal"
  # access_level 自动设置，无需传入
}

# 获取知识库列表
GET /api/knowledge-bases?page=1&page_size=12&scope_type=personal

# 获取知识库详情
GET /api/knowledge-bases/{kb_uuid}

# 更新知识库
PUT /api/knowledge-bases/{kb_uuid}
Body: {
  "name": "更新后的名称",
  "description": "更新后的描述"
}

# 删除知识库
DELETE /api/knowledge-bases/{kb_uuid}
```

#### 文档管理

```http
# 上传文档
POST /api/kb-documents/{kb_uuid}
Content-Type: multipart/form-data
Body: {
  "file": (binary),
  "auto_vectorize": true
}

# 获取文档列表
GET /api/kb-documents/{kb_uuid}?page=1&page_size=20

# 开始向量化
POST /api/kb-documents/{kb_uuid}/{doc_uuid}/vectorize

# 删除文档
DELETE /api/kb-documents/{kb_uuid}/{doc_uuid}
```

#### 智能体关联管理

```http
# 获取可用知识库列表（未关联的）
GET /api/knowledge-bases/agents/{agent_uuid}/available-knowledge-bases

# 获取已关联知识库列表
GET /api/knowledge-bases/agents/{agent_uuid}/knowledge-bases

# 批量添加知识库关联
POST /api/knowledge-bases/agents/{agent_uuid}/knowledge-bases/batch
Body: [
  {
    "knowledge_base_uuid": "kb-001",
    "priority": 10,
    "is_enabled": true
  },
  {
    "knowledge_base_uuid": "kb-002",
    "priority": 8,
    "is_enabled": true
  }
]

# 更新关联配置
PUT /api/knowledge-bases/agents/{agent_uuid}/knowledge-bases/{kb_uuid}
Body: {
  "priority": 9,
  "is_enabled": true
}

# 删除关联
DELETE /api/knowledge-bases/agents/{agent_uuid}/knowledge-bases/{kb_uuid}
```

#### 检索 API

```http
# 向量检索
POST /api/knowledge-bases/{kb_uuid}/search
Body: {
  "query": "勾股定理是什么",
  "top_k": 5,
  "similarity_threshold": 0.7,
  "search_mode": "hybrid"
}

# 智能体检索（多知识库）
POST /api/knowledge-bases/agents/{agent_uuid}/retrieve
Body: {
  "query": "如何学好数学"
}
```

### 常见问题

#### Q1: 如何选择知识库类型？

**A**: 问自己一个问题："谁应该能看到这个知识库？"
- 只有我自己 → **个人级**
- 我们学校的师生 → **学校级**
- 特定课程的师生 → **课程级**
- 平台所有用户 → **系统级**（需管理员权限）

#### Q2: 优先级如何设置？

**A**: 建议值：
- **10** = 最重要（课程核心知识）
- **8-9** = 重要（补充资料）
- **5-7** = 一般（参考资料）
- **1-4** = 次要（背景知识）

不确定就用系统自动分配的！

#### Q3: 支持哪些文档格式？

**A**: 目前仅支持：
- ✅ Markdown (.md)
- ✅ 纯文本 (.txt)

这两种格式轻量、易于处理、无需复杂解析。

#### Q4: 知识库可以修改类型吗？

**A**: 可以，但需要注意：
- 修改类型会改变可见范围
- 例如：个人级 → 学校级，从"仅自己"变为"本校师生可见"

#### Q5: 智能体可以使用多少个知识库？

**A**: 理论上没有限制，但建议：
- 3-5个为佳（太多会增加检索时间）
- 合理设置优先级
- 可以临时禁用不常用的
